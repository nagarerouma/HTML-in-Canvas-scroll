Scrollable Notes

この面は、HTML の長文コンテンツをそのまま立方体へ貼り付けるサンプルです。背景の黄色は body 側で描画し、その上に半透明の文章パネルを重ねています。

重要なのは、スクロールする主体を body ではなく .leadtext にしている点です。これにより、index.html 側で「今ホイールしている場所にある scroll container」を見つけやすくなります。

この段落はスクロール確認用です。ホイールを回すと、この面の中だけが上下に動きます。スクロール可能な間は立方体ズームよりも面内スクロールを優先します。

HTML-in-Canvas では、見た目は WebGL の 3D 面でも、元データは普通の DOM です。つまり、文字、背景、ボーダー、スクロールバーは通常の CSS で調整できます。

カスタムスクロールバーも .leadtext に直接付けています。macOS のオーバーレイスクロールバー環境では標準のままだと非常に見えにくいので、このサンプルでは太さと色を明示しています。

この文章は高さを十分に作るためのダミーです。黄色い背景が見えるように、パネルは四辺に余白を残して配置しています。面全体が単色に潰れず、立方体上でも色が分かりやすくなります。

クリックしても、この面では特別な action は起きません。button や input のような interactive 要素が無い面では、index.html 側が click / focus 転送を行わないようにしています。

そのため、以前のように「面をクリックしたら文字が消える」問題は、この静的テキスト面では発生しないはずです。静的面は静的面として扱い、スクロールだけを許可します。

さらに確認したい場合は、この面を単体で開いても同じようにスクロールできます。立方体上では wheel の転送が 1 段追加されるだけで、中身の HTML 自体は普通のページです。

以下はスクロール量を十分に作るための補助テキストです。長文が続くことで scrollbar が常時必要になり、見た目と動作を確認しやすくなります。

スクロールバーのトラックには明るい黄色、つまみには濃い金色を使っています。背景と近い色相にしつつも、明度差を付けて見失いにくくしています。

立方体の面に貼られた HTML は、見た目だけが 3D になっているだけで、iframe の中では通常の CSS レイアウトが成立しています。したがって、overflow や scrollbar の制御も通常の Web ページと同じ発想で設計します。

ただし実際の入力は canvas 側が受けるため、ホイールやクリックは index.html から明示的に転送する必要があります。このサンプルでは、スクロール可能な要素が見つかったときだけ wheel をその要素へ流します。

もしスクロール可能な要素が見つからなければ、従来どおり立方体全体のズームとして扱います。つまり、scroll と zoom は競合ではなく、優先順位付きの処理に分けています。

背景色についても、body 全体に描いたグラデーションが面の土台です。その上に余白付きの半透明パネルを重ねているため、背後の黄色がちゃんと見えます。

この構成にすると、面の色が完全に隠れず、かつ文字の可読性も保てます。全面を不透明な白パネルにすると読みやすくても、立方体の面ごとの個性が消えてしまいます。

また、非インタラクティブ面への click 転送を止めたことで、静的テキストを誤って focus 対象にしないようにしています。これが「クリックしたら文字が消える」症状への対策です。

この段落以降も同じ目的で、あえて文章量を増やしています。サンプルとしては冗長ですが、scrollbar の視認性テストには十分な高さが必要です。

スクロール中に cube 側のズームが始まってしまう場合は、wheel 判定が scroll container を見つけられていないことになります。今回の修正ではその経路も見直しています。

HTML-in-Canvas の挙動はブラウザフラグや実装状態に左右されるため、単体ページと立方体上の両方で確認するのが安全です。この面はその確認用として使いやすい構成にしてあります。

読み終わる頃には、パネル右端の scrollbar がはっきり見えるはずです。もし見えない場合は、まず overflow が発生しているか、次に macOS のシステム設定の影響を疑うのが順当です。

このサンプルでは CSS 側で十分に太いスクロールバーを指定しているので、少なくとも WebKit 系ブラウザではトラックとつまみの両方が確認できるはずです。

最後の確認用テキストです。ここまででまだ 1 画面に収まるようなら、iframe の高さが想定より大きい環境です。その場合でも、この追加入力で確実にスクロール領域になるようにしています。

最後の段落です。ここまで読めたら、スクロール領域、背景色、文字表示、クリック時の安定性の 4 点がまとめて確認できます。
